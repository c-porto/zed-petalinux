Vivado 2023.1 Docker Image
====================================

.. contents:: Table of Contents
   :local:

Prerequisites
-------------
The Dockerfile assumes that the Web Installer for AMD Vivado was downloaded and 
copied to this directory. The installation does work with the full compressed 
tar installer and, in fact, will allow the image to built **much** quicker (the
web installer based one will take at least one hour), however since the 
Dockerfile copies the full installer to the build environment the disk usage
will be at least 200 GB on build.

Authentication
~~~~~~~~~~~~~~
With the web installer based build it is necessary to authenticate with AMD 
Xilinx to download the Vivado payloads. The installer handles that using an 
auth token that can be generated by the installer itself. To create the token
run the following commands.

.. code-block:: bash

    ./Xilinx_Unified_2023.<version>_Lin64.bin --keep --noexec --target ./xinstall

This will extract the installer so that is possible to access the ``xsetup`` 
script. With the installer extracted run:

.. code-block:: bash

    ./xsetup -b AuthTokenGen

This command will prompt you to enter your Xilinx credentials to generate the 
token, which will be copied to ``$HOME/.Xilinx/wi_authentication_key``. With 
the token ready, copy it to this directory and then proceed to the build 
section.
    
Building
--------
To build the image just run:

.. code-block:: bash

    docker build -t vivado:2023.1 .

This will take some time, since downloads from Xilinx servers are capped to 
3 MB/sec of speed.

Running the Container
---------------------

To start the container just run the following command:

.. code-block:: bash

    docker run -it --rm --network host -e DISPLAY=$DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix -v <flatsat-project-dir>:/home/vivado/workdir vivado:2023.1
